The trace runs on the Python engine on the server; the browser only renders the streamed grid snapshots.

New to SSDs? This page shows, live, what happens inside a solid-state drive as a workload writes to it — how data lands in flash blocks, how stale data piles up, and how garbage collection and wear unfold over time. Switch to ZNS mode to see zoned namespaces instead.

Each box is one zone (a group of blocks written strictly sequentially). The bar shows the zone's pages: valid (live) stale (invalidated) unwritten (below write pointer)  —  border: EMPTY OPEN FULL. ● hot / ● cold mark the open streams; the number is the zone's reset count.
Each square is one flash block; the green fill shows its invalid (stale) page ratio, exactly as in conventional mode. The device still runs its own garbage collection — but a colored border marks the block's placement handle (reclaim unit): hot (overwrites) cold (first writes). Because same-lifetime data shares a reclaim unit, GC victims are mostly-invalid together → lower write amplification.
